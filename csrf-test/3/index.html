🎯 CSRF 취약점 테스트 #3

VTSP - 견적 요청서 등록

⚠️ 중요: CSRF 토큰 무시됨!

원본 패킷에 _csrf 토큰이 포함되어 있지만, 서버에서 토큰 검증을 하지 않거나 잘못 구현되어 있어 토큰 없이도 요청이 성공합니다. 이것이 바로 CSRF 취약점입니다!

⚠️ 취약점 정보

타입: Cross-Site Request Forgery (CSRF)
영향도: Critical - 사용자 동의 없이 견적 요청서 등록 가능
원인: CSRF 토큰 검증 미흡, SameSite 쿠키 미설정

📋 공격 대상 정보

대상 URL: https://cyberts.kr/vtsp/mrk/estmt/request/insert.do
메소드: POST
변경 파라미터: estmtNm (견적명)
기타 파라미터: 14개 (자동 입력, CSRF 토큰 제거됨)

📖 테스트 방법

  1. cyberts.kr에 로그인한 상태에서 이 페이지를 엽니다
  2. "CSRF 공격 실행" 버튼을 클릭합니다
  3. cyberts.kr의 견적 요청 내역으로 이동합니다
  4. 견적명이 입력한 이름으로 등록되었는지 확인합니다
  5. 증명 완료: 본인이 직접 등록하지 않았는데 견적 내역이 생겼다면 CSRF 취약점 증명!
